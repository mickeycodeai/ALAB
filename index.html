ALAB Platform — Release Notice

Document Reference: ALAB-RN-2026-003
Effective Date: 28 June 2026
Distribution: Field Operations / Research Personnel

This release transitions the ALAB vector surveillance platform from a client-side processing architecture to a centralized, server-side inference model. All analytical workloads are now handled by dedicated backend services, resulting in improved detection consistency, reduced device-side resource consumption, and a more auditable processing pipeline across all active modules.

Visual Specimen Analysis Module

The visual detection subsystem has been re-engineered to offload all image inference to a server-hosted detection service. Upon capture, the system encodes the current frame and transmits it to the designated inference endpoint for processing. Results are returned as structured detection records containing specimen count, spatial localization data, and per-detection confidence ratings. Mean confidence across the detection set is computed and surfaced to the operator following each scan cycle.

This revision eliminates the previous dependency on a third-party cloud workflow provider and removes all browser-resident computer vision libraries from the client bundle. Application initialization time is reduced, and detection behavior is now governed exclusively by the deployed server-side model rather than heuristic client-side logic.

Acoustic Species Identification Module

The audio-based species classification module continues to operate via the centralized inference service, now routed through a dedicated acoustic analysis endpoint separate from the visual pipeline. This separation improves fault isolation between modules and allows each subsystem to be maintained or updated independently without service interruption.

Live microphone recordings and uploaded audio files are both processed through the same classification pathway. The module produces a species and sex determination alongside a confidence rating and a full probability distribution across all supported vector categories. The classification decision is derived directly from the model output, with no supplementary rule-based adjustment applied.

System Architecture and Operational Integrity

Visual and acoustic inference services are now decoupled at the endpoint level, each operating as an independent backend process. Surveillance data relay to the operations server — including specimen counts, session timestamps, and GPS coordinates — remains active and unmodified, with payload authentication unchanged from the prior release.

All ancillary platform modules have been verified operational under the updated architecture, including geospatial mapping, field position acquisition, session log export, manual image submission, and device hardware integration. No functional changes have been made to these components in this release.

Platform Status

All core modules are active and operational. This release is approved for field deployment. Operators encountering connectivity issues with the inference service should verify backend availability before initiating a scan session.

Prototype Software — Not for Clinical Use

This application is an early-stage research prototype. Detection accuracy may vary due to hardware limitations including camera resolution, lens quality, focal distance, lighting conditions, and device processing speed. Results should not be used as a sole basis for public health decisions.

Republic Act No. 10173 — Data Privacy Act of 2012

What We Collect

This application collects GPS coordinates and mosquito egg counts solely for the purpose of dengue vector surveillance. No personal information — including your name, contact details, identity, or device identifiers — is collected or stored at any time.

How Data Is Used

Anonymized location and count data is transmitted to a secure research server to generate field surveillance maps. All transmitted records are non-attributable and cannot be linked back to any individual user.

Data Storage & Retention

Captured images are processed locally on your device and are never uploaded to any external server. Only aggregate numerical counts and approximate GPS coordinates are transmitted. Data is retained only for the duration of the research period.

Your Rights Under RA 10173

As a data subject under Philippine law, you have the right to be informed, to object, and to erasure of any data linked to you. Because no personal data is collected, these rights are satisfied by design. For concerns, contact the system administrator.

Consent

By tapping "I Understand & Accept," you confirm that you have read this notice, understand the prototype limitations of this software, and voluntarily consent to the anonymous collection of GPS and count data as described above in accordance with RA 10173.
